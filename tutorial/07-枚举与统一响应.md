## 7.1 枚举类型 enum

枚举用于定义**固定数量的常量**，比常量更安全。

### 定义枚举

```java
// enums/Gender.java
package com.tutorial.bootdemo.enums;

import lombok.AllArgsConstructor;
import lombok.Getter;

@Getter  // 生成 get 方法
@AllArgsConstructor  // 生成全参构造
public enum Gender {
    BOY("boy", "1"),   // 男：详情是"boy"，数据库存"1"
    GIRL("girl", "2"); // 女：详情是"girl"，数据库存"2"

    private String detail;  // 显示用
    private String dbValue; // 存储用
}
```

### 为什么用枚举？

```java
// ❌ 不用枚举：String 容易写错
private String gender;  // "boy", "girl", "Boy", "BOY"... 各种写法

// ✅ 用枚举：类型安全，只能是 BOY 或 GIRL
private Gender gender;  // 只能是 Gender.BOY 或 Gender.GIRL
```

### 枚举的方法

```java
Gender gender = Gender.BOY;

gender.getDetail();   // "boy"
gender.getDbValue();  // "1"
gender.name();        // "BOY"（枚举名称）

// 根据数据库值获取枚举
Gender.fromValue("1");  // → Gender.BOY
```

## 7.2 统一响应封装

为什么需要统一响应格式？

```java
// ❌ 不统一：每个接口返回格式都不一样
{"id": 1, "name": "张三"}
{"code": 200, "data": {...}}
{"status": "ok", "result": {...}}

// ✅ 统一：所有接口返回格式一致
{
  "success": true,
  "data": {...},
  "errorMsg": null
}
```

## 7.3 泛型 Response 类

```java
// Response.java
public class Response<T> {  // <T> 泛型：可以是 StudentDTO、List、Long...

    private T data;        // 成功时返回的数据
    private boolean success;  // 是否成功
    private String errorMsg;  // 错误信息

    // 成功响应
    public static <K> Response<K> newSuccess(K data) {
        Response<K> response = new Response<>();
        response.setData(data);
        response.setSuccess(true);
        return response;
    }

    // 失败响应
    public static Response<Void> newFail(String errorMsg) {
        Response<Void> response = new Response<>();
        response.setErrorMsg(errorMsg);
        response.setSuccess(false);
        return response;
    }

    // getters and setters...
}
```

### 泛型说明 `<T>`

```java
// 返回单个学生
Response<StudentDTO> → {"data": {...}}

// 返回学生列表
Response<List<StudentDTO>> → {"data": [...]}

// 返回 ID
Response<Long> → {"data": 1}

// 无数据返回
Response<Void> → {"data": null}
```

## 7.5 异常处理

### Service 层抛出异常

```java
// 查询不到抛异常
Student student = studentRepository.findById(id)
        .orElseThrow(() -> new RuntimeException("学生不存在"));

// 业务校验抛异常
if (CollectionUtils.isEmpty(list)) {
    throw new IllegalStateException("邮箱已被使用");
}
```

### Controller 层处理

```java
@GetMapping("/student/{id}")
public Response<StudentDTO> getStudentById(@PathVariable long id) {
    try {
        return Response.newSuccess(studentService.getStudentById(id));
    } catch (RuntimeException e) {
        return Response.newFail(e.getMessage());
    }
}
```

### 全局异常处理（进阶）

可以用 `@ControllerAdvice` 统一处理所有异常，详见后续课程。
